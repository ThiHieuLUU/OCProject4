4 ~~~
5  
6 The `ast27` module helps Python applications to process trees of the Python
7 abstract syntax grammar. The abstract syntax itself might change with
8 each Python release; this module helps to find out programmatically what
9 the current grammar looks like and allows modifications of it. The `ast27`
10 module is similar to the builtin `ast` module on Python 2.7, except `ast27`
11 runs on Python 3 and provides PEP 484 type comments as part of the AST.
12  
13 Specifically, these changes are made to the Python 2.7 AST:
14 - The `FunctionDef`, `Assign`, `For`, and `With` classes all have a
15 `type_comment` field which contains a `str` with the text of the
16 associated type comment, if any.
17 - `arguments` has a `type_comments` list of per-argument type comments.
18 - `parse` has been augmented so it can parse function signature types when
19 called with `mode=func_type`.
20 - `Module` has a `type_ignores` field which contains a list of
21 lines which have been `# type: ignore`d.
22 - `Str` has a `kind` string field which preserves the original string
23 prefix, so that `ast27.parse('br"test"').body[0].value.kind == 'br'`.
24  
25 An abstract syntax tree can be generated by using the `parse()`
26 function from this module. The result will be a tree of objects whose
27 classes all inherit from `ast27.AST`.
28  
29 A modified abstract syntax tree can be compiled into a Python code object
30 using the built-in `compile()` function.
31  
32 Additionally various helper functions are provided that make working with
33 the trees simpler. The main intention of the helper functions and this
34 module in general is to provide an easy to use interface for libraries
35 that work tightly with the python syntax (template engines for example).
36
